# Normalização de dados

A normalização é o processo de organizar os dados em um banco de dados para reduzir a redundância (repetição) e melhorar a integridade dos dados. Em termos simples, é a técnica de "dividir tabelas" para garantir que cada dado seja armazenado em apenas um lugar. 

Aqui estão as três primeiras formas normais (as mais usadas no dia a dia):
1. Primeira Forma Normal (1FN) 
Uma tabela está na 1FN se não houver grupos repetidos ou valores múltiplos em uma única célula. 
Regra: Cada coluna deve ter valores atômicos (indivisíveis) e cada registro deve ser único.
Exemplo: Em vez de uma coluna Telefone com três números separados por vírgula, você cria registros separados ou uma tabela específica para telefones. 
2. Segunda Forma Normal (2FN)
Para estar na 2FN, a tabela já deve estar na 1FN e todos os atributos que não são chaves devem depender da chave primária total. 
Regra: Eliminar dados que se aplicam apenas a uma parte de uma chave composta.
Exemplo: Se você tem uma tabela de Itens de Pedido, o nome do produto não deve estar lá (pois depende do ID do Produto, não do ID do Pedido). O nome do produto vai para a tabela Produtos. 
3. Terceira Forma Normal (3FN)
Para estar na 3FN, a tabela deve estar na 2FN e nenhum atributo pode depender de outro atributo que não seja a chave (eliminar dependências transitivas). 
Regra: Um campo não pode depender de outro campo que, por sua vez, depende da chave.

Exemplo: Na tabela Clientes, você tem CEP e Cidade. Como a Cidade depende do CEP (e o CEP depende do ID do Cliente), você deve mover Cidade e Estado para uma tabela de endereços separada. 

## Por que normalizar?

Economia de espaço: Menos repetição de texto.
Evita anomalias: Se um cliente mudar de endereço, você altera em um só lugar, e não em centenas de registros de vendas.
Organização: O modelo fica mais lógico e fácil de entender. 

Dica: Nem sempre a normalização máxima é o ideal. Em sistemas de Big Data ou relatórios (Data Warehousing), às vezes "desnormalizamos" propositalmente para ganhar velocidade de leitura.

Segue um script SQL em **MySQL** ilustrando o processo de normalização de um **sistema de vacinação** passando por **1FN → 2FN → 3FN**, com comentários para uso em sala de aula.

### **1. Banco de dados e tabela inicial (desnormalizada)**

```sql
-- Criação do banco (opcional)
CREATE DATABASE IF NOT EXISTS sistema_vacinacao;
USE sistema_vacinacao;

-- Tabela inicial DESNORMALIZADA (antes da 1FN)
CREATE TABLE IF NOT EXISTS vacinas_desnormalizadas (
    paciente VARCHAR(100),
    cpf VARCHAR(14),
    vacinas TEXT,         -- múltiplas vacinas separadas por ; (NÃO-ATÔMICO)
    doses  TEXT,          -- múltiplas doses separadas por ; (NÃO-ATÔMICO)
    clinicas TEXT         -- múltiplas clínicas separadas por ; (NÃO-ATÔMICO)
);

-- Insert para exemplo (para ilustrar 1FN)
INSERT INTO vacinas_desnormalizadas (paciente, cpf, vacinas, doses, clinicas)
VALUES 
    ('Ana Silva', '111.222.333-44', 'COVID;Gripe;HPV', '1;2;1', 'Posto A;Posto B;Posto A'),
    ('João Pereira', '555.666.777-88', 'Gripe;Tetra', '1;2', 'Posto C;Posto A');
```

### **2. 1FN – Primeira Forma Normal (toda célula atômica)**

```sql
-- 1FN: cada combinação de vacina/dose/clínica em uma linha única
CREATE TABLE IF NOT EXISTS aplicacoes_vacina_1fn (
    id INT AUTO_INCREMENT PRIMARY KEY,
    paciente VARCHAR(100),
    cpf VARCHAR(14),
    vacina VARCHAR(50),
    dose INT,
    clinica VARCHAR(50)
);

-- Dados normalizados (Você pode explicar que este é o "explode" manual da desnormalizada)
INSERT INTO aplicacoes_vacina_1fn (paciente, cpf, vacina, dose, clinica) VALUES
    ('Ana Silva',   '111.222.333-44', 'COVID', 1, 'Posto A'),
    ('Ana Silva',   '111.222.333-44', 'Gripe', 2, 'Posto B'),
    ('Ana Silva',   '111.222.333-44', 'HPV',   1, 'Posto A'),
    ('João Pereira','555.666.777-88', 'Gripe', 1, 'Posto C'),
    ('João Pereira','555.666.777-88', 'Tetra', 2, 'Posto A');
```

### **3. 2FN – Segunda Forma Normal (eliminar dependência parcial)**

```sql
-- 2FN: separar PACIENTE (depende só de CPF) do histórico de aplicação

-- Tabela de PACIENTES (1 entidade)
CREATE TABLE IF NOT EXISTS pacientes (
    id_paciente INT AUTO_INCREMENT PRIMARY KEY,
    cpf VARCHAR(14) UNIQUE NOT NULL,
    paciente VARCHAR(100)
);

-- Tabela de VACINAS (opcional, se quiser FKS)
CREATE TABLE IF NOT EXISTS vacinas (
    id_vacina INT AUTO_INCREMENT PRIMARY KEY,
    nome_vacina VARCHAR(50) NOT NULL
);

-- Tabela de registo de aplicação (2FN) com FKs
CREATE TABLE IF NOT EXISTS aplicacoes_vacina_2fn (
    id_aplicacao INT AUTO_INCREMENT PRIMARY KEY,
    id_paciente INT NOT NULL,
    id_vacina INT NOT NULL,
    dose INT,
    clinica VARCHAR(50),
    FOREIGN KEY (id_paciente) REFERENCES pacientes(id_paciente),
    FOREIGN KEY (id_vacina) REFERENCES vacinas(id_vacina)
);

-- Popular tabelas filhos (só para exemplo)
INSERT INTO pacientes (cpf, paciente) VALUES
    ('111.222.333-44', 'Ana Silva'),
    ('555.666.777-88', 'João Pereira');

INSERT INTO vacinas (nome_vacina) VALUES
    ('COVID'),
    ('Gripe'),
    ('HPV'),
    ('Tetra');

-- Converter dados 1FN → 2FN
INSERT INTO aplicacoes_vacina_2fn (id_paciente, id_vacina, dose, clinica)
SELECT 
    p.id_paciente,
    v.id_vacina,
    av.dose,
    av.clinica
FROM aplicacoes_vacina_1fn av
JOIN pacientes p ON p.cpf = av.cpf
JOIN vacinas v ON v.nome_vacina = av.vacina;
```

### **4. 3FN – Terceira Forma Normal (eliminar dependência transitiva)**

```sql
-- 3FN: criar tabela CLINICAS, pois GESTOR depende de CLINICA, não de id_aplicacao

CREATE TABLE IF NOT EXISTS clinicas (
    id_clinica INT AUTO_INCREMENT PRIMARY KEY,
    clinica VARCHAR(50) NOT NULL UNIQUE,
    gestor VARCHAR(100)
);

CREATE TABLE IF NOT EXISTS aplicacoes_vacina_3fn (
    id_aplicacao INT AUTO_INCREMENT PRIMARY KEY,
    id_paciente INT NOT NULL,
    id_vacina INT NOT NULL,
    dose INT,
    id_clinica INT NOT NULL,
    FOREIGN KEY (id_paciente) REFERENCES pacientes(id_paciente),
    FOREIGN KEY (id_vacina) REFERENCES vacinas(id_vacina),
    FOREIGN KEY (id_clinica) REFERENCES clinicas(id_clinica)
);

-- Cadastrar clínicas com gestores
INSERT INTO clinicas (clinica, gestor) VALUES
    ('Posto A', 'Dr. Maria'),
    ('Posto B', 'Dr. João'),
    ('Posto C', 'Dr. Ana');

-- Converter 2FN → 3FN (usando clinica como chave de junção)
INSERT INTO aplicacoes_vacina_3fn (id_paciente, id_vacina, dose, id_clinica)
SELECT 
    a2.id_paciente,
    a2.id_vacina,
    a2.dose,
    c.id_clinica
FROM aplicacoes_vacina_2fn a2
JOIN clinicas c ON c.clinica = a2.clinica;
```
