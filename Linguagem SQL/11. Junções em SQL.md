# Junções

Junções em SQL combinam dados de duas ou mais tabelas baseadas em colunas relacionadas, como chaves estrangeiras. São essenciais para consultas reais em bancos relacionais, como ligar "Colaborador" a "Departamento".[^1][^2]

## Tipos Principais de JOIN

Existem quatro tipos comuns, representados por diagramas de Venn (interseção para INNER, tudo da esquerda para LEFT etc.). Use INNER para matches exatos; OUTER para incluir sem correspondência.


| Tipo | Descrição | Quando Usar |
| :-- | :-- | :-- |
| INNER JOIN | Só linhas com correspondência em ambas tabelas  | Análises precisas com dados relacionados  |
| LEFT JOIN | Todas da esquerda + matches da direita (NULL se sem match)  | Listar tudo da tabela principal  |
| RIGHT JOIN | Todas da direita + matches da esquerda (raro, evite)  | Similar a LEFT, mas invertido  |
| FULL OUTER JOIN | Tudo das duas (NULL onde sem match)  | Relatórios completos (nem todos SGBDs suportam)  |

## Sintaxe Básica

Sempre: `SELECT colunas FROM tabela1 JOIN tabela2 ON tabela1.id = tabela2.id_chave`.

Exemplo com tabelas Colaborador (id, nome, id_depto) e Departamento (id, nome_depto):

```sql
-- INNER JOIN: só colaboradores com depto
SELECT c.nome, d.nome_depto 
FROM Colaborador c 
INNER JOIN Departamento d ON c.id_depto = d.id;[web:33]
```


## Exemplos Práticos

Considere:

- Colaborador: (1, 'Ana', 10), (2, 'João', 20), (3, 'Maria', NULL)
- Departamento: (10, 'TI'), (20, 'RH'), (30, 'Vendas')

| JOIN | Exemplo SQL | Resultado Esperado |
| :-- | :-- | :-- |
| INNER | `SELECT c.nome, d.nome_depto FROM Colaborador c INNER JOIN Departamento d ON c.id_depto = d.id` [^4] | Ana-TI, João-RH (Maria excluída) [^1] |
| LEFT | `SELECT c.nome, d.nome_depto FROM Colaborador c LEFT JOIN Departamento d ON c.id_depto = d.id` [^5] | Ana-TI, João-RH, Maria-NULL [^2] |
| RIGHT | `SELECT c.nome, d.nome_depto FROM Colaborador c RIGHT JOIN Departamento d ON c.id_depto = d.id` [^6] | Ana-TI, João-RH, NULL-Vendas [^5] |
| FULL | `SELECT c.nome, d.nome_depto FROM Colaborador c FULL OUTER JOIN Departamento d ON c.id_depto = d.id` [^7] | Ana-TI, João-RH, Maria-NULL, NULL-Vendas [^8] |

## JOIN com Múltiplas Tabelas

Encadeie JOINs para 3+ tabelas:

```sql
SELECT c.nome, d.nome_depto, pr.nome AS projetista
FROM Colaborador c
INNER JOIN Departamento d ON c.id_depto = d.id
LEFT JOIN Projeto pr ON c.id = pr.id_colab;[web:36]
```
